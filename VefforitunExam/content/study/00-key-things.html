Everything You Need to Know

Every topic explained — what it is, why it matters, how it works, and what to watch out for.

1. Flexbox

Flexbox lays items out along one axis — either a row or a column. You set display: flex on a container and its direct children become flex items. Not grandchildren — only one level down.

There are two axes. When flex-direction: row (the default), the main-axis runs horizontally and the cross-axis runs vertically. When flex-direction: column, they swap.

justify-content controls how items are spaced along the main-axis. Values: flex-start (default), center, flex-end, space-between, space-around.

align-items controls how items are positioned along the cross-axis. Default is stretch — items fill the full height. Other values: flex-start, center, flex-end, baseline.

align-content controls how flex lines (not items) are distributed. It only has any effect when flex-wrap: wrap is set and there are multiple lines. Without wrap, it does nothing.

flex-grow controls how items grow to fill remaining space. Default is 0 (don't grow). The remaining space is divided proportionally:

Container: 750px. Three items: 100px each.
flex-grow: 1, 1, 3 → total = 5

Remaining: 750 - 300 = 450px
1 unit = 450 / 5 = 90px

Item 1: 100 + 90×1 = 190px
Item 2: 100 + 90×1 = 190px
Item 3: 100 + 90×3 = 370px

flex-shrink is the opposite — when items overflow, they shrink proportionally. Default is 1. Set to 0 to prevent shrinking.

flex-basis sets the initial size before growing/shrinking. When both flex-basis and width are set, flex-basis wins.

float, clear, and vertical-align have NO effect on flex items.

flex-flow is shorthand for flex-direction + flex-wrap: flex-flow: row wrap

align-self overrides align-items for a single item.

order changes the visual order of an item. Default: 0.

flex shorthand: flex: 0 1 auto is the default (grow shrink basis). flex: 1 = flex: 1 1 0% — the item grows and shrinks equally.

2. CSS Grid

Grid is a two-dimensional layout system — it handles rows AND columns at the same time. Flexbox is one-dimensional (row OR column). Use Grid for page-level layouts (header, sidebar, main, footer). Use Flexbox for component internals (cards, navbars).

grid-template-areas defines named regions:

.page {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 250px 1fr;
}

The fr unit represents a fraction of the available space. 1fr 2fr = first column gets 1/3, second gets 2/3.

repeat(auto-fit, minmax(250px, 1fr)) creates a responsive grid without media queries — fits as many columns as possible.

Grid terminology: a grid line is a dividing line (numbered from 1). A grid track is a full row or column. A grid cell is a single box. A grid area is a rectangular group of cells. All areas must be rectangular or the layout breaks.

grid-column: 1 / 3 means "start at line 1, end at line 3" (spans 2 columns). span 2 is shorthand for "span 2 tracks".

grid-gap (or gap) sets spacing between grid items.

Grid items ignore float, clear, and vertical-align — same as flexbox.

3. BEM

BEM is a CSS naming convention: Block__Element--Modifier.

A Block is a standalone entity (e.g., .card). An Element is a part of a block, separated by double underscore (e.g., .card__title). A Modifier is a variation, separated by double dash (e.g., .card__title--active).

Rules: lowercase letters, digits, and dashes only. No camelCase. No PascalCase. Blocks use class selectors only — no tag names, no IDs.

4. CSS Variables

CSS Variables (custom properties) let you store reusable values. Define with -- prefix, use with var():

/* From teacher's code: summer-vacation-css3/styles/base.css */
:root {
  --main-blue-color: #63B4D1;
}
.nav {
  background-color: var(--main-blue-color);
}

They are scoped — define on :root for global, or on a specific selector for local. They can be updated dynamically with JavaScript using element.style.setProperty('--color', 'red'). They cascade down to descendants.

They do NOT replace CSS preprocessors like SASS entirely — preprocessors have mixins, nesting, and functions that CSS variables alone cannot do.

5. Tailwind CSS

A utility-first CSS framework — instead of writing CSS classes in a separate file, you apply utility classes directly in your HTML: bg-blue-500 text-white px-4 py-2 rounded.

How it works: scans your HTML/JS files for class names and generates only the styles you actually use. Benefits: no context-switching between files, no naming fatigue, delete a component and its styles disappear automatically.

Modifiers use prefix + colon: hover:bg-red-500, md:flex, dark:text-white.

6. Box Sizing

content-box (default): width/height apply to content only. Padding and border are added on both sides on top of that. So width: 100px + padding: 10px×2 + border: 10px×2 = 140px total.

border-box: padding and border are included inside the width. Total is always exactly what you set. Content area shrinks to make room. Width: 100px = 100px total. Content becomes 60px.

8. Prototypes & Inheritance

JavaScript objects can inherit from other objects through the prototype chain. When you access a property, JS looks on the object first, then its prototype, then that prototype's prototype, up to Object.prototype.

The teacher shows two ways to define methods:

// Basic Constructor — each instance gets its OWN copy (wastes memory)
function Car(model) {
  this.model = model;
  this.toString = function() { return this.model; };  // new function per instance
}

// Constructor With Prototypes — ONE shared function (memory efficient)
function Car(model) {
  this.model = model;
}
Car.prototype.toString = function() { return this.model; };  // shared by all

Inheritance pattern (from teacher's shapes.js):

function Shape(position) { this.position = position; }
Shape.prototype.render = function() {};

function Rectangle(position, width, height) {
    Shape.call(this, position);    // 1. Run parent's setup on this child
    this.width = width;
    this.height = height;
}
Rectangle.prototype = Object.create(Shape.prototype);  // 2. Link prototype chain
Rectangle.prototype.constructor = Rectangle;            // 3. Fix constructor reference

Step 1 copies properties. Step 2 links methods. Step 3 is cleanup — without it, Rectangle.prototype.constructor would point to Shape instead of Rectangle.

Objects are mutable and passed by reference (from slides). var a = obj — both point to the SAME object. Changing a.x changes obj.x.

Property attributes (from slides): every property has writable, enumerable, configurable. All default to true for regular assignment, but false when using Object.defineProperty(). This is why built-in methods like toString don't show in for/in loops — they're non-enumerable.

Testing properties: hasOwnProperty() = own only (not inherited). "prop" in obj = own AND inherited. o.hasOwnProperty("toString") → false (inherited), but "toString" in o → true.

ES6 class is syntactic sugar over prototypes — same thing underneath. extends replaces Object.create(Parent.prototype). super() replaces Parent.call(this).

9. Promises

A Promise represents a value that doesn't exist yet — it will either resolve (success) or reject (failure) in the future.

.then() runs on success. .catch() runs on failure. .finally() runs either way.

The Promise constructor runs synchronously. Only the .then() callback is async (microtask).

resolve() does NOT stop the executor — code after it keeps running:

new Promise((resolve) => {
    console.log('A');     // runs immediately
    resolve('B');         // marks as resolved, doesn't stop
    console.log('C');     // still runs
});
// A, C print now. B goes to microtask queue.
MethodResolves whenRejects when
Promise.all()ALL resolveANY one rejects (immediately)
Promise.race()First to SETTLE (resolve or reject)First to SETTLE
Promise.any()First to RESOLVEALL reject (AggregateError)
Promise.allSettled()ALL settle (always)Never rejects

Promise.allSettled returns: { status: 'fulfilled', value: ... } or { status: 'rejected', reason: ... }. Check promise.status === 'fulfilled' to tell them apart.

for await...of (from teacher's Visual Promises): processes promises ONE AT A TIME, in order. Each resolves before the next starts. If one rejects, the loop stops.

10. Event Loop

JavaScript is single-threaded. The Event Loop is how it handles async operations without blocking.

When an async operation (setTimeout, fetch) is called, the browser handles it via Web APIs. When it's done, the callback goes into a queue. The Event Loop checks: if the call stack is empty, process the queue.

Two queues, strict priority:

  1. ALL microtasks first (Promise .then, queueMicrotask)
  2. Then ONE macrotask (setTimeout, setInterval)
  3. Back to step 1
console.log('1');                          // sync → 1
setTimeout(() => console.log('2'), 0);     // macrotask
Promise.resolve().then(() => console.log('3')); // microtask
console.log('4');                          // sync → 4
// Output: 1, 4, 3, 2

async/await: Syntactic sugar over Promises. Code before await runs synchronously. await pauses the function and returns control to the caller. Code after await runs as a microtask.

fetch API: fetch() returns a Promise. It resolves even on 404/500 — only rejects on network errors. You MUST check response.ok. And response.json() returns a Promise — you must await it.

11. TypeScript

TypeScript adds types to JavaScript. It catches errors before you run the code. Types are stripped out during compilation — the output is plain JavaScript.

Type inference: TypeScript can guess types without you writing them: let x = 3 → TypeScript knows x is a number.

Interfaces: Define a shape that objects must follow. interface User { name: string; age: number; }

Union types: A value can be multiple types: string | number (pipe = "or")

Partial<T> makes all properties optional. Type assertion = value as Type. Nullable = string | null.

Structural typing: "Type checking focuses on the shape the values have" (from slides). If an object has the required properties, it satisfies the type — extra properties are OK.

Compilation — two things happen (from slides): (1) type annotations are stripped out, (2) modern syntax is "downleveled" to older JS (e.g., template literals become .concat()). Default target is ES3.

Errors don't stop output (from slides): "Although TypeScript generates errors, we still see an output in our generated .js files." TypeScript is built for gradual migration — you can adopt it one file at a time.

Generics (from slides): "provide variables to types." Example: Array<number> constrains an array to only contain numbers. Without generics, arrays could contain anything.

Duck typing (from slides): structural typing is "commonly referred to as duck typing" — if it has the right shape, it satisfies the type.

Strictness: "strict": true in tsconfig.json enables noImplicitAny, noImplicitThis, strictFunctionTypes, etc. You can opt out of individual checks.

12. React

A component is a function that accepts props and returns JSX describing UI. Components should be: Single Responsibility, Pure/Predictable, Reusable & Composable, Encapsulated.

useState — holds state. The setter triggers a re-render. Use functional updater setCount(prev => prev + 1) when new state depends on previous — this avoids stale closures.

useEffect — runs side effects after render.

  • useEffect(fn, []) — once on mount
  • useEffect(fn, [x]) — mount + when x changes
  • useEffect(fn) — every render (usually a bug!)
  • Return a cleanup function: return () => clearInterval(id) — runs before unmount and before re-run

Stale closure: Inside useEffect with [], variables are captured at mount time. setCount(count + 1) always reads the initial value (0). Fix: setCount(prev => prev + 1).

useRef — a mutable value that persists across renders but does NOT trigger re-renders. Good for: DOM refs, timer IDs, previous values. NOT for: triggering renders, sharing state between components.

useCallback — caches a function reference between re-renders. Prevents child components (wrapped in React.memo) from re-rendering unnecessarily.

useMemo — caches a computed value. Avoids expensive recalculations on every render. "By default React will re-run the entire body of your component every time it re-renders" (from slides).

Rules of Hooks: (1) Only call at the top level — never inside if/loops/nested functions. (2) Only from React function components or custom hooks.

13. Context API

React's built-in way to share state without passing props through every level (solves prop drilling).

Three steps: createContext() creates it, Provider wraps the tree and holds the value, useContext() reads it in any child.

// Source: workshops/w5/context/cart-context.tsx
const CartContext = createContext(null);

export function CartProvider({ children }) {
  const [cart, setCart] = useState([]);
  function addItem(item) { setCart((prev) => [...prev, item]); }
  function removeItem(id) { setCart((prev) => prev.filter((p) => p.id !== id)); }
  return (
    <CartContext.Provider value={{ cart, addItem, removeItem }}>
      {children}
    </CartContext.Provider>
  );
}

export function useCart() {
  const ctx = useContext(CartContext);
  if (!ctx) throw new Error("useCart must be used within CartProvider");
  return ctx;
}

Downside: When the context value changes, ALL consumers re-render — even if they only use part of the value. Bad for frequently changing state.

Next.js gotcha: Context providers use useState, so they MUST have "use client" at the top. A Context provider is always a Client Component.

14. Zustand

A tiny state management library. No Provider needed. No boilerplate. The store IS a hook.

// Source: workshops/w5/store/cart-store.ts
export const useCartStore = create()(
  persist(
    (set) => ({
      cart: [],
      addItem: (item) => set((state) => ({ cart: [...state.cart, item] })),
      removeItem: (id) => set((state) => ({
        cart: state.cart.filter((p) => p.id !== id)
      })),
    }),
    { name: "cart-storage", storage: createJSONStorage(() => localStorage) }
  )
);

// Usage — anywhere, no Provider needed:
const { cart, addItem } = useCartStore();

persist middleware saves state to localStorage automatically. Three main middlewares: persist, devtools, immer.

Selectors prevent re-renders: Use useCartStore((state) => state.cart) to select specific values — NOT const { cart, addItem } = useCartStore() which re-renders on ANY change. For computed selectors that return new arrays, use useShallow for shallow comparison.

Zustand requires immutable updates (unlike Redux which has Immer). You must spread or create new references manually: set((state) => ({ cart: [...state.cart, item] })).

15. Redux Toolkit

State management with structure. The data flows in one direction: View → Action → dispatch() → Reducer → Store → View.

createSlice bundles a reducer + action creators. It uses Immer — you write code that looks like mutation (state.cart.push(item)) but Immer produces a new immutable state behind the scenes.

// Source: workshops/w5/store/cart-slice.ts
const cartSlice = createSlice({
  name: "cart",
  initialState: { cart: [] },
  reducers: {
    addItem(state, action) {
      state.cart.push(action.payload);  // looks like mutation — Immer makes it safe
    },
    removeItem(state, action) {
      state.cart = state.cart.filter((p) => p.id !== action.payload);
    },
  },
});
export const { addItem, removeItem } = cartSlice.actions;

configureStore sets up the store — automatically combines reducers, adds Redux Thunk middleware, and enables DevTools.

4 concepts to know: Connected Components (linked via useSelector/useDispatch), Actions (plain objects: {type, payload}), Reducers (pure functions: (state, action) → newState), Dispatch (sends actions to the store).

TypeScript: Use PayloadAction<Product> to type action payloads. Export RootState and AppDispatch types from the store for typed hooks.

Provider pattern in Next.js: Redux needs <Provider store={store}> wrapping the app — this must be in a "use client" component. The teacher creates a separate providers.tsx file (from w5/app/providers.tsx).

17. Next.js Routing

Next.js uses file-system routing: folders = URL segments. A page.tsx file makes a route publicly accessible.

Special files: layout.tsx (shared UI, persists across navigation), loading.tsx (loading UI, wraps page in Suspense), error.tsx (error boundary, must be "use client"), not-found.tsx (404 page), route.ts (API endpoint — cannot coexist with page.tsx).

Component hierarchy: Layout → Template → Error → Loading → Page.

Dynamic routes: [slug] = one segment. [...slug] = catch-all (1+ segments). [[...slug]] = optional catch-all (0+). Route groups (name) organize without adding URL segments.

Prefetching: <Link> auto-prefetches routes when they enter the viewport. <a> does not.

18. Server vs Client Components

Server Component (default in App Router): runs on the server. No JavaScript sent to the browser. CAN: fetch data, access databases, read env vars. CANNOT: use hooks (useState, useEffect), handle events (onClick), use browser APIs.

Client Component: add "use client" at the top of the file. Runs in the browser. CAN: use hooks, handle events, access browser APIs. CANNOT: access databases or server resources directly.

Server Components reduce the JavaScript bundle size. They can render Client Components as children. The boundary is the "use client" directive.

19. Server Action vs API Route

Server Action = an async function marked with "use server". It runs on the server and can be called directly from a Client Component or form. Use for: form submissions, data mutations, revalidating data.

API Route = an HTTP endpoint defined in route.ts. Use when an external service needs to call your backend: Stripe webhooks, mobile apps, public REST APIs.

The decision rule: Who calls it? Your React app → Server Action. External service → API Route.

// Source: concert-arena/actions/booking.ts

export async function submitBooking(data) {
  const bookingRef = crypto.randomUUID();
  return { success: true, bookingRef };
}

20. react-hook-form

A form library that uses uncontrolled inputs — the form doesn't re-render the component on every keystroke. It only re-renders on validation events. This is its primary advantage over managing forms with useState.

Key methods: useForm() initializes, register("field") connects an input, handleSubmit(fn) wraps the submit handler, formState.errors has the validation errors.

Set mode: "onChange" to validate as the user types. Use resolver: zodResolver(schema) to integrate Zod for schema validation.

Valid built-in rules: required, minLength, maxLength, pattern, validate. NOT valid: type, sanitize.

Why server-side validation is still needed: client validation can be bypassed entirely via curl, DevTools, or disabling JavaScript. The server must always re-validate.

Multi-step forms (from teacher's concert-arena): FormProvider goes in the layout.tsx ("use client"). Each step is a separate route page using useFormContext(). Use trigger(["field1", "field2"]) to validate per-step before advancing. Use getValues() on the review page. Zod schemas are composed with .merge().

Controller — for non-native inputs (MUI, custom components). register() only works with native HTML inputs. For custom UI: <Controller control={control} name="field" render={({ field }) => ...} />

Controlled vs Uncontrolled: Controlled = useState + onChange (re-renders every keystroke). Uncontrolled = refs (no re-render). react-hook-form uses uncontrolled — that's why it's fast.

Server Actions with forms (alternative to react-hook-form): Function signature is (prevState, formData: FormData) => Promise. Extract data with formData.get("fieldName"). Use with useActionState hook for pending state.

21. DOM

The DOM is a tree. The highest node is the document. From any node you can access parentNode, childNodes, nextSibling, previousSibling.

children vs childNodes: children returns only elements. childNodes returns everything including text nodes (whitespace between tags). This is why nextSibling often returns a text node — use nextElementSibling to get the next element.

querySelectorAll() is the most powerful selector — accepts any CSS selector. Returns a static NodeList (snapshot). getElementsByClassName() returns a live HTMLCollection that updates when the DOM changes.

innerHTML renders HTML markup. textContent renders as plain text. From the slides: textContent "renderast ekki sem HTML heldur strengur".

Creating elements: createElement() creates, appendChild() adds to end, insertBefore(new, ref) adds before a reference node, removeChild() removes (called on parent), replaceChild(new, old) replaces.

Events: addEventListener(event, handler) attaches a handler. Events bubble up from the target to the root. stopPropagation() stops bubbling. preventDefault() prevents default behavior (form submit, link navigation).

22. CSS3

Gradients: linear-gradient() defaults to top→bottom. radial-gradient() radiates from center outward. Can stack multiple gradients (first = on top).

box-shadow: offset-x offset-y blur-radius spread-radius color. inset keyword puts the shadow inside. Multiple shadows (comma-separated) can replicate elements.

Transitions only activate when a property changes (e.g., on :hover). Shorthand: transition: property duration timing-function delay. Animations run independently via @keyframes — they don't need a trigger.

Transforms: rotate(), scale(), skew(), translate(). transform-origin defaults to 50% 50% (center). Multiple transforms are space-separated and ORDER MATTERS — rotate then translate is different from translate then rotate.

Perspective: CSS property on parent = shared vanishing point for all children. Transform value on element = each element gets its own vanishing point.

border-radius: 4 values = TL TR BR BL (clockwise). 50% on a square = circle. Slash syntax: 20px / 100% — horizontal radius / vertical radius (elliptical corners).

background-size: cover fills the container (may crop). contain fits inside (may leave gaps). Multiple backgrounds: first listed = on top.

23. Selectors

Attribute selectors: [attr] has it, [attr="val"] equals, [attr^="val"] begins with, [attr$="val"] ends with, [attr*="val"] contains, [attr~="val"] contains word.

:first-child = first child of parent. The element must BE the first child. :first-of-type = first of its type among siblings — finds the first <p> even if a <div> comes before it.

:nth-child(an+b) — 2n+1 = odd, 2n = even, 3 = exactly the 3rd. Counts ALL children. :nth-of-type() counts only the same type. :not(selector) = negation — everything that does NOT match.

::before / ::after insert generated content. The content property is MANDATORY — without it, nothing renders. ::first-line styles the first line. ::first-letter styles the first letter (drop caps). attr() inserts an attribute value: content: " (" attr(href) ") ".

Combinators: A B = anywhere inside (descendant). A > B = direct child only. A + B = immediately after (adjacent sibling). A ~ B = any sibling after (general sibling).

24. Socket.io

Enables real-time, bidirectional communication between client and server. Unlike HTTP (request-response), Socket.io keeps a persistent connection open.

MethodWho receives
io.emit()ALL clients
socket.emit()Server: back to sender. Client: to server.
socket.broadcast.emit()ALL EXCEPT sender
io.to(room).emit()All in that room

Rooms: socket.join(roomId) adds a socket to a room. Used for chat channels/groups. io.to(room).emit() sends to everyone in the room.

Middleware: io.use((socket, next) => { ... }) runs before every connection. Used for authentication. Call next() to allow, next(new Error()) to reject.

Listening: socket.on(event, callback) listens for events on both client and server.

26. Web APIs

Web Components: customElements.define('tag-name', Class) registers a custom HTML element. Must extend HTMLElement. Tag names must contain a hyphen. Shadow DOM (attachShadow({mode: "open"})) encapsulates CSS — styles don't leak in or out. connectedCallback() fires when added to DOM. Use <template> elements for the HTML structure, clone with templateContent.cloneNode(true). Read attributes with getters: get imageUrl() { return this.getAttribute("image-url"); }

Web Workers: Run JavaScript in a background thread. Cannot access the DOM. Communicate with the main thread via postMessage(). Good for CPU-intensive calculations.

Drag and Drop: Set draggable="true" on the element. Handle dragstart (set data), dragover (must preventDefault() or drop won't work), and drop (get data). Use dataTransfer.setData() / getData().

localStorage: Persistent key-value storage. Survives browser close. ~5MB limit. Same-origin only. API: setItem(key, value), getItem(key). Must JSON.stringify() objects.

sessionStorage: Same API as localStorage but cleared when the tab closes.
