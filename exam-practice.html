Flexbox (from slides)
TermDefinition (from slides)
FlexboxMakes the once-difficult task of laying out your page, widget, application, or gallery almost simple. Provides functionality for equal height columns which is easy to implement
display: flexIn order to create a flex container you must apply the display CSS property with the value flex. Every nested elements are a part of that flow
flex containerWhen an element is a flex container it will only flex its immediate children, and not further descendants. These descendants can be made as flex containers as well
Axis systemFlexbox works on an axis grid system. Flex items are laid along the main-axis. Flex lines are added in the direction of the cross-axis
flex-directionValues: row (default), row-reverse, column, column-reverse
flex-wrapIf flex-items do not fit within the flex container we need to control that. By default flex containers don't wrap. Values: nowrap (default), wrap, wrap-reverse. Height of line is determined by tallest/widest element
flex-flowflex-flow combines both flex-direction and flex-wrap in a shorthand syntax
justify-contentIt gives us control on how flex items in a flex line are distributed along the main-axis. Values: flex-start (default), center, space-around, space-between, flex-end
align-itemsThe align-items CSS property defines how flex items are aligned along it's flex line's cross-axis. Values: flex-start, center, baseline, stretch (default), flex-end
align-contentalign-content aligns a flex container's lines within a flex container that has extra space in the cross-axis direction. Only works with wrap. Values: stretch (default), flex-start, center, space-around, space-between, flex-end
flex-growDefines whether a flex item is allowed to grow when there is available space. Determines how much proportionally relative to siblings. Accepts non-negative numbers. Default: 0
Growth factor750px container, 3 items at 100px, grow values 1+1+3=5 total. Remaining 450px / 5 = 90px/unit. Item 1&2: 100+90=190px. Item 3: 100+270=370px
flex-shrinkDetermines how much a flex item will shrink relative to siblings when there isn't enough space. Defines how negative space is distributed. Default: 1
flex-basisDefines the initial or default size of flex items, before extra or negative space is distributed. Accepts same values as width. When both flex-basis and width are set, basis trumps width. Default: auto
flex (shorthand)Shorthand for flex-grow, flex-shrink and flex-basis. Example: flex: 1 0 100%
align-selfUsed to override the align-items property on a per-flex-item basis. Default: auto
orderProvides more granular control of visual ordering. Accepts integer. Default: 0. Ignored on non flex items
No effect on flex itemsfloat, clear, and vertical-align don't have an effect on flex items. absolute positioning takes them out of the flow

CSS3: Transforms
TermDefinition
transformCSS property to apply 2D or 3D transformations to elements
rotate()rotate(45deg) — rotates around Z-axis. Same as rotateZ(45deg)
rotateX() / rotateY()Rotates around X-axis (tilts forward/back) or Y-axis (tilts left/right). 3D effect requires perspective
rotate3d()rotate3d(x, y, z, angle) — rotate around a custom axis vector
scale()scale(2) — doubles size. scaleX(2) / scaleY(2) / scaleZ(2) / scale3d(1.5, 2, 2)

Multiple transformsSpace-separated (not comma!): transform: rotate(180deg) translateY(20px). Order matters! rotate(180deg) then translateY(20px) moves UP because Y-axis is flipped
CSS3: Perspective
TermDefinition
perspective (property)CSS property on the parent/container. Creates a shared 3D space for all children. Applies to all child elements. Example: .wrapper { perspective: 1000px; }
perspective (transform value)Used inside transform on the element itself. Only applies to that element. Example: transform: perspective(500px) rotateX(50deg)
Key differenceProperty = shared vanishing point for all children (realistic cards fanning out). Transform value = each element has its own vanishing point (all look identical)
perspective-originSets the viewpoint position. Default: 50% 50%. Takes x-position (left/right/center/%) and y-position (top/bottom/center/%)
Depth valueLower value = more dramatic 3D effect. Higher value = subtler. Defaults to 0
